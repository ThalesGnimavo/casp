# 26-06-17-003 — casp-help: `casp help` + per-command help

Fixes a papercut and adds command discoverability on top of the 0.6.0 bundle
(`26-06-17-002`). Tooling ergonomics only — no protocol change, no LLM, no
telemetry, no network.

## What shipped

- **`casp help` is first-class.** It prints the top-level overview and exits 0.
  Previously the most natural thing a user types (`casp help`) fell through to
  the unknown-command path and exited 1.
- **Per-command help.** `casp help <command>` and `casp <command> --help` print
  a focused block for that verb — a one-line "why it exists", usage, every flag,
  and 1-2 real examples. All 11 verbs covered (`init`, `status`, `check`,
  `next`, `new`, `ship`, `close`, `install-hook`, `verify`, `state`, `help`).
  The two entry points render byte-identical output.
- **"How the loop works" in the overview.** A short `THE LOOP` section —
  `init → status → (work) → check → ship/close → push` — names the mental model
  in four lines, with `check` called out as the only hard gate.
- **Graceful unknown commands.** `casp help <bogus>` and `casp <bogus>` both
  print `no such command: <name>`, list the valid verbs, point at
  `casp help <command>`, and exit 1. No stack trace, no full-help dump.

## Design

- One structured registry (`src/help.ts` `COMMANDS`) is the source of truth.
  `COMMAND_NAMES` (the valid-command list) and the per-command blocks read from
  it, so the unknown-command list and the help surface cannot name different
  verbs. The top-level block stays hand-grouped for scanning but pulls its verb
  canon from the same array.
- `src/cli.ts` `main()` dispatch restructured: version first, then no-command /
  `-h` / `--help` → overview, then `help` → `runHelp`, then per-command
  `--help` / `-h` interception (`rest.includes(...)`, exact match — narrower
  than the prior `args.includes('--help')`), then the existing `switch`. The old
  inline `HELP` const was deleted; the `default:` case now uses the shared
  `unknownCommandMessage`.
- CLI help is usage-focused: one tight "what is CASP" paragraph plus a pointer
  to casp.sh. The binary ships a frozen snapshot; the site stays current.

## Tests

- 63 total (9 new): `casp help` exit 0 and identical to the no-arg / `-h` /
  `--help` block; the `THE LOOP` section present; `help check` is check-specific
  and `check --help` matches it byte-for-byte; every verb has a focused block
  reachable both ways; `help <bogus>` and `<bogus>` exit 1 with the
  valid-command list; `-V` / `--version` still short-circuit. All green.
